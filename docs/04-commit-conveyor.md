# Конвейер коммитов

Как делать status → add → commit → push и убедиться, что коммит публичный.

## Цель

- Понимать базовый цикл: проверить изменения, добавить в индекс, закоммитить, отправить на GitHub.
- Убедиться, что коммит попал в публичную историю.

## Шаги

1. **Проверка состояния:**
   ```bash
   git status
   git diff
   ```
   Увидишь изменённые и неотслеживаемые файлы.

2. **Добавление в индекс:**
   ```bash
   git add путь/к/файлу
   # или все изменённые:
   git add -A
   ```
   Ещё раз `git status` — файлы должны быть в «Changes to be committed».

3. **Коммит:**
   ```bash
   git commit -m "Краткое сообщение: что сделано"
   ```

4. **Отправка на GitHub:**
   ```bash
   git push
   ```
   При первом push может понадобиться: `git push -u origin main` (если ветка называется `main`).

## Проверка, что коммит публичный

- Зайди на страницу репозитория на GitHub.
- Открой вкладку **Commits** (или главную страницу — видно последний коммит).
- Убедись, что твой коммит и сообщение отображаются.

## Типовые ошибки

- **Nothing added to commit:** нужно выполнить `git add` перед `git commit`.
- **Push rejected:** см. гайд [Устранение неполадок](10-troubleshooting.md).

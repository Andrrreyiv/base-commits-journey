# Workflow CHANGELOG

Как вести CHANGELOG и когда добавлять записи.

## Цель

- Понимать структуру CHANGELOG (Unreleased, версии, секции).
- Добавлять записи к коммитам или релизам без суеты.

## Структура

- **[Unreleased]** — изменения, ещё не вошедшие в релиз. Секции: Добавлено / Изменено / Исправлено / Удалено.
- При релизе: блок Unreleased переименовывают в ** [X.Y.Z] - YYYY-MM-DD**, ниже добавляют новый пустой **[Unreleased]** с секциями.

## Когда добавлять запись

- После значимого коммита (новая фича, гайд, скрипт, исправление) — добавь пункт в Unreleased.
- Можно накапливать несколько пунктов и дописать блок перед релизом.

## Как добавить запись вручную

Открой CHANGELOG.md, найди секцию `## [Unreleased]`, под ней подсекцию (например `### Добавлено`) и добавь строку:

```markdown
- Краткое описание изменения.
```

## Хелпер-скрипт

Из корня репо:

```bash
python scripts/add_changelog_entry.py "Добавлено" "Описание пункта"
```

Скрипт вставит пункт в Unreleased. Секция и текст передаются аргументами.

## Проверка

- CHANGELOG читается сверху вниз; Unreleased в начале; даты у версий заполнены при релизе.

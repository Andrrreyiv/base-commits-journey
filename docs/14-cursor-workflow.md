# Работа в Cursor

Как работать с агентом, как смотреть diff, что не коммитить.

## Цель

- Использовать Cursor (и агента) для правок и коммитов осознанно.
- Перед коммитом видеть diff и не включать лишнее.

## Как работать с агентом

- Формулируй задачу чётко: «добавь гайд по X», «исправь ссылку в README».
- Проверяй предложенные изменения: открой diff (Source Control или сравнение файлов) перед принятием.
- Не принимай массовые правки без просмотра — агент может затронуть не те файлы.

## Как смотреть diff

- **Source Control (Ctrl+Shift+G):** список изменённых файлов, клик по файлу — diff.
- **Командная строка:** `git diff` — все изменения, `git diff путь/к/файлу` — один файл.
- Перед коммитом: `git status` и `git diff --stat` — что попадёт в коммит.

## Что не коммитить

- Секреты: `.env`, токены, пароли.
- Временные файлы: логи, кэш, `__pycache__/` (должны быть в .gitignore).
- Лишние правки: если агент изменил 20 файлов, а нужен один — добавь в коммит только нужный: `git add путь/к/файлу`.

## Рекомендации

- Один коммит — одна мысль. Разбивай большие изменения на несколько коммитов.
- После правок агента: `git status`, `git diff`, затем `git add` только нужное и `git commit`.

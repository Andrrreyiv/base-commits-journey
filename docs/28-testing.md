# Гайд по тестам

Как запускать тесты и как устроена папка tests/.

## Цель

- Запускать юнит-тесты локально и понимать вывод.
- Добавлять тесты для новых скриптов.

## Запуск

Из корня репозитория:

```bash
python -m unittest discover -v -s tests
```

В CI тесты запускаются автоматически при push и PR.

## Структура

- `tests/` — все тесты (только stdlib, unittest).
- `tests/test_*.py` — модули с тестами (по скрипту или группе).
- `tests/fixtures/` — тестовые файлы при необходимости.

## Добавление теста

1. Создай `tests/test_имя_скрипта.py`.
2. Импортируй модуль из `scripts/` (добавь корень в `sys.path` при необходимости).
3. Пиши тесты в классе, наследующем `unittest.TestCase`.
4. Запусти `python -m unittest discover -v -s tests` и убедись, что всё зелёное.

## Проверка

- Все тесты проходят: в конце выводится «OK» и число тестов.
- Упавший тест: в выводе виден traceback и имя теста.

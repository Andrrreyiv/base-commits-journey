# FAQ

Частые вопросы и короткие ответы.

## 1. Сколько коммитов нужно для гильдии?

Зависит от правил гильдии. Часто указывают число публичных коммитов на default-ветке (например, 100). Смотри описание гильдии и чеклист [Верификация](12-guild-verify-checklist.md).

## 2. Считаются ли merge-коммиты?

Да. Коммиты, попавшие в default-ветку (в том числе через merge), учитываются.

## 3. Можно ли делать пустые коммиты (--allow-empty)?

Для цели «качественная история» — не рекомендуется. Гильдия может не засчитывать такие коммиты или это может противоречить правилам.

## 4. Git не распознаётся в терминале после установки.

Перезапусти терминал. Если не помогло — добавь путь к Git в PATH (например, `C:\Program Files\Git\cmd`).

## 5. Authentication failed при push.

Часто из-за 2FA: нужен Personal Access Token вместо пароля. См. [Аутентификация по HTTPS](05-auth-https-gcm.md).

## 6. Как отвязать remote и привязать другой репо?

`git remote remove origin` затем `git remote add origin новый-URL`.

## 7. Push rejected (non-fast-forward).

На GitHub есть новые коммиты. Выполни `git pull --rebase origin main` (или `git pull`), затем `git push`.

## 8. Где смотреть число коммитов?

На GitHub: репо → Commits или счётчик на главной. Локально: `git rev-list --count HEAD`.

## 9. Репозиторий приватный — коммиты засчитаются?

Обычно нет. Для «public commits» репо должен быть публичным.

## 10. Как исправить последний коммит (сообщение или файлы)?

Сообщение: `git commit --amend -m "Новое сообщение"`. Файлы: измени файлы, затем `git add` и `git commit --amend --no-edit`. После amend при уже запушенном коммите понадобится `git push --force` (осторожно в общих ветках).

## 11. Что такое PAT?

Personal Access Token — токен на GitHub вместо пароля, нужен при включённой 2FA. Создаётся в Settings → Developer settings → Personal access tokens.

## 12. Нужен ли SSH, если уже работает HTTPS?

Нет. HTTPS с сохранёнными учётными данными (или PAT) достаточно. SSH — альтернатива для тех, кто предпочитает ключи.

# Чеклист верификации гильдии

Connect GitHub, verify, что проверить перед verify.

## Цель

- Подключить GitHub-аккаунт к гильдии и пройти верификацию.
- Убедиться, что всё готово до нажатия «Verify».

## Что проверить перед верификацией

1. **Публичный репозиторий:** репо, по которому верифицируешь, должен быть **Public**.
2. **Default-ветка:** коммиты должны быть на default-ветке (обычно `main`). Проверь на GitHub: главная страница репо → ветка по умолчанию.
3. **Достаточно коммитов:** если гильдия требует N коммитов — на вкладке **Commits** (или счётчик на главной) должно быть не меньше N.
4. **Ты — автор коммитов:** имя/email в коммитах соответствуют аккаунту GitHub, который подключаешь к гильдии (проверь `git config user.name` и `user.email`).

## Подключение GitHub

- В настройках гильдии (или на странице верификации) нажми **Connect GitHub** / **Link GitHub**.
- Разреши доступ к аккаунту GitHub (OAuth).
- После подключения станет доступна кнопка **Verify** (или аналогичная).

## Что проверять после Verify

- Статус верификации (успех / ошибка).
- Если ошибка — прочитай сообщение: часто указывают «not enough commits» или «repository is private»; исправь и нажми Verify снова.

## Типовые причины неудачи

- Репозиторий приватный — сделай Public.
- Мало коммитов — добавь коммиты на default-ветку.
- Подключён другой аккаунт GitHub — отвяжи и привяжи нужный.

# Стиль сообщений коммитов

Шаблоны сообщений, плохие и хорошие примеры.

## Цель

- Понятные сообщения коммитов: что сделано и зачем (в одном предложении).
- Единый стиль в репозитории.

## Шаблоны

- **Docs:** `Docs: краткое описание` — документация.
- **Chore:** `Chore: краткое описание` — настройки, конфиги, без изменения логики.
- **CI:** `CI: краткое описание` — изменения в workflow, скриптах CI.
- **Scripts:** `Scripts: краткое описание` — скрипты в `scripts/`.
- **Fix:** `Fix: что исправлено` — исправление бага.

Первая строка — до 50–72 символов. При необходимости вторая строка пустая, далее тело коммита.

## Хорошие примеры

- `Docs: add goal and rules guide`
- `Chore: add .editorconfig for consistent formatting`
- `CI: wire link checker into workflow`
- `Fix: correct URL in README`

## Плохие примеры

- `update` — неясно что обновлено.
- `fix` — неясно что исправлено.
- `WIP` — лучше не коммитить незавершённое в main.
- Очень длинная первая строка без смысла.

## Проверка

- `git log -1 --oneline` — сообщение читается и отражает изменение.

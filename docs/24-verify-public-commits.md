# Verified email и учёт contributions

Почему verified email важен и как GitHub считает вклад в граф.

## Цель

- Понять, зачем верифицировать email на GitHub.
- Убедиться, что коммиты попадают в граф contributions.

## Verified email

- В настройках GitHub (Settings → Emails) должен быть **verified** тот email, который указан в `git config user.email`.
- Коммиты с неверифицированным email не отображаются в зелёном графе на профиле и могут не учитываться гильдией.

## Как верифицировать

1. GitHub → Settings → Emails.
2. Добавь email (тот же, что в `user.email`), нажми **Verify**.
3. Перейди по ссылке из письма.

## Как считаются contributions

- Учитываются коммиты в **публичных** репозиториях на **default-ветке** (или в ветках, влитых в default).
- Коммиты с **verified** email и датой в последний год отображаются в графе.
- Merge-коммиты и коммиты в форках (если PR принят в upstream) тоже считаются.

## Проверка

- На профиле GitHub граф заполнен за последние коммиты.
- В настройках Emails у используемого адреса стоит галочка Verified.

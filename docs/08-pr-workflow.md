# Workflow Pull Request

Как создать PR, пройти ревью, смержить и связать с issue.

## Цель

- Создать ветку, внести изменения, отправить ветку на GitHub и открыть Pull Request.
- Понимать базовый цикл: ревью → правки (при необходимости) → merge.

## Шаги

1. Создай ветку от актуального `main` и переключись на неё:
   ```bash
   git switch main
   git pull
   git switch -c feature/краткое-название
   ```

2. Внеси изменения, закоммить, отправь ветку:
   ```bash
   git add -A
   git commit -m "feat: описание"
   git push -u origin feature/краткое-название
   ```

3. На GitHub открой репозиторий — появится кнопка **Compare & pull request** для только что отправленной ветки.
4. Заполни заголовок и описание PR. Если задача связана с issue, укажи в описании `Closes #номер` или «Fixes #номер».
5. Нажми **Create pull request**. После ревью (или самопроверки) нажми **Merge pull request**.

## После merge

- Переключись на `main` и подтяни изменения:
  ```bash
  git switch main
  git pull
  ```
- Локальную ветку можно удалить: `git branch -d feature/краткое-название`.

## Связка с issue

- В описании PR: `Closes #5` — после merge issue #5 автоматически закроется.
- В коммитах можно писать «Fix #5» в сообщении — GitHub свяжет коммит с issue.

## Типовые ошибки

- **Merge conflict:** нужно разрешить конфликты в файлах, затем `git add` и завершить merge/rebase (см. [Устранение неполадок](10-troubleshooting.md)).
- **PR не виден:** убедись, что ветка запушена: `git push -u origin имя-ветки`.

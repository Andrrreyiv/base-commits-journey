# Ежедневный workflow

Как организовать день, чтобы делать по 1–2 осмысленных коммита без спешки.

## Цель

- Выделять 15–30 минут в день на коммит.
- Не накапливать изменения: один день — одна-две темы.
- Держать историю чистой и атомарной.

## Утро или начало сессии

1. **Обнови репозиторий:** `git pull` (если работаешь с нескольких мест).
2. **Проверь прогресс:** `python tools/base_commit_coach.py --report` — число коммитов и записей в PROGRESS.
3. **Подсказка по задаче:** `python tools/base_commit_coach.py --next` — что делать следующим по плану.

## Выбор задачи на день

- Открой [PROGRESS.md](../PROGRESS.md) и [ROADMAP.md](../ROADMAP.md).
- Выбери одну задачу: документ, скрипт, тест или правку в CI.
- Добавь в PROGRESS новую строку с ID (следующий P-XXX), датой и критериями приёмки.

## Работа и коммит

1. Внеси изменения (один логический шаг).
2. Запусти локальные проверки: `make check` или скрипты из `scripts/`.
3. `git status` → `git add` только нужные файлы.
4. `git commit -m "Краткое сообщение из плана"`.
5. `git push`.

## После коммита

- Заполни в PROGRESS: Files changed, Status = Done, Notes.
- При желании обнови CHANGELOG (Unreleased) через `scripts/add_changelog_entry.py` или вручную.

## Если нет времени на задачу

- Не коммить пустышку. Лучше пропустить день.
- Можно только обновить PROGRESS или документацию — это уже полезный коммит, если есть что описать.

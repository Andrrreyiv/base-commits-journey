# Как пользоваться шаблонами (issue и PR)

Где лежат шаблоны, как их заполнять и когда выбирать какой.

## Цель

- Использовать шаблоны issue и PR при создании задач и запросов на слияние.
- Заполнять чеклисты и описания так, чтобы ревью было проще.

## Шаблоны issue

- Лежат в `.github/ISSUE_TEMPLATE/`: например `bug_report.md`, `feature_request.md`.
- При **New issue** GitHub предлагает выбрать тип: Bug report или Feature request.
- В шаблоне уже есть заголовок и поля: описание, шаги воспроизведения, окружение (для бага) или зачем нужно (для фичи).
- Заполни все релевантные блоки; лишнее можно удалить.

## Шаблон PR

- Файл `.github/pull_request_template.md` — его содержимое подставляется в описание каждого нового PR.
- Обычно там: краткое описание, чеклист (что проверил), как воспроизвести, связь с issue (Closes #N).
- Отмечай галочки чеклиста по мере выполнения; не удаляй пункты без причины.

## Конфиг шаблонов issue

- `.github/ISSUE_TEMPLATE/config.yml` — отключает «blank» issue (без шаблона) и может добавлять ссылки на документацию и FAQ.
- Если нужен ещё один тип issue — добавь новый `.md` в эту папку с полем `name:` и т.д.

## Проверка

- Создай черновик issue (не публикуй) — убедись, что выбран нужный шаблон и поля понятны.
- Открой новый PR — в описании должен быть шаблон с чеклистом.

## Типовые ошибки

- **Оставил в issue «Шаг 1 / Шаг 2» без текста** — заполни или удали, иначе ревьюер не поймёт задачу.
- **В PR не отметил чеклист** — отметь выполненные пункты перед запросом ревью.

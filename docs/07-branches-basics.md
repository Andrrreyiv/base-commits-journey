# Основы веток

Зачем нужны ветки, когда создавать, как переключаться (switch/checkout).

## Цель

- Понимать, что ветка — это линия коммитов.
- Уметь создавать ветку и переключаться на неё без потери работы.

## Шаги

1. **Текущая ветка:**
   ```bash
   git branch --show-current
   ```
   Обычно это `main` или `master`.

2. **Создать новую ветку и переключиться:**
   ```bash
   git switch -c feature/название
   ```
   Или по-старому: `git checkout -b feature/название`.

3. **Переключиться на существующую ветку:**
   ```bash
   git switch main
   ```
   Или: `git checkout main`.

4. **Список веток:**
   ```bash
   git branch -a
   ```

## Когда создавать ветку

- Новая фича или эксперимент — отдельная ветка, чтобы не ломать `main`.
- Исправление бага — можно ветку `fix/описание` или править в `main`, если репо маленький и один работаешь.

## Безопасный workflow

- Перед переключением ветки: `git status` — если есть незакоммиченные изменения, закоммить или `git stash`.
- Не переключайся с несохранёнными изменениями в конфликтующих файлах — Git предупредит.

## Типовые ошибки

- **«Please commit your changes»:** закоммить или `git stash`, затем переключиться.
- **Ветка «потерялась»:** она только локальная; `git branch -a` покажет все ветки, включая удалённые.

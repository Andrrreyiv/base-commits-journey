# Глоссарий

Краткие определения терминов.

- **repo (репозиторий)** — хранилище истории Git: коммиты, ветки, теги. Может быть локальным или на сервере (например, GitHub).

- **remote** — связь с другим репозиторием (часто на GitHub). Пример: `origin`.

- **origin** — стандартное имя remote, указывающее на «свой» репозиторий на GitHub (clone откуда делали или куда пушим).

- **branch (ветка)** — линия коммитов с именем. Обычно `main` или `master` — основная ветка.

- **commit (коммит)** — снимок изменений с сообщением и автором. Идентифицируется хешем (SHA).

- **push** — отправить свои коммиты на remote (например, на GitHub).

- **pull** — забрать коммиты с remote и подтянуть в текущую ветку (часто pull = fetch + merge).

- **PR (Pull Request)** — предложение влить ветку в другую (например, в `main`) на GitHub. После ревью делают merge.

- **merge** — объединение веток: коммиты одной ветки попадают в другую.

- **clone** — скопировать репозиторий с remote (например, с GitHub) к себе на компьютер.

- **default branch** — ветка по умолчанию в репо (часто `main`). На неё открывается главная страница репо на GitHub.

- **public repo** — публичный репозиторий: любой может просматривать код и коммиты.

- **PAT (Personal Access Token)** — токен доступа на GitHub вместо пароля; используется при 2FA.

- **2FA (two-factor authentication)** — двухфакторная аутентификация. На GitHub при 2FA для Git нужен PAT.

# Как считаются публичные коммиты

Где смотреть commits/main, public vs private.

## Цель

- Понять, где GitHub и гильдия смотрят число коммитов.
- Различать публичные и приватные репозитории.

## Где смотреть число коммитов

- **На GitHub:** открой репозиторий → вкладка **Commits** или главная страница (под описанием репо) — отображается количество коммитов на выбранной ветке (обычно `main`).
- **Локально:** `git rev-list --count HEAD` — число коммитов в текущей ветке до HEAD.
- **На странице профиля:** GitHub показывает «X contributions» за период; для гильдии обычно важны коммиты в **публичном** репозитории на **default-ветке** (часто `main`).

## Public vs private

- **Публичный репозиторий:** коммиты в нём видны всем; они учитываются в «public» статистике.
- **Приватный репозиторий:** коммиты не видны; гильдия и сторонние сервисы обычно считают только публичные репо.
- Для цели «100 public commits» нужен именно **публичный** репозиторий и коммиты на default-ветке.

## Default-ветка

- Обычно это `main` или `master`. Указана на главной странице репо под списком веток.
- Считаются коммиты именно в этой ветке (и влитые в неё через merge).

## Итог

Публичные коммиты = коммиты в публичном репозитории на default-ветке. Смотри число на GitHub (Commits) или считай локально `git rev-list --count main` после `git fetch origin`.

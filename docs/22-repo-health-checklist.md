# Чеклист здоровья репозитория

Что проверить перед верификацией гильдии или релизом.

## Цель

- Убедиться, что репо в порядке: ссылки, структура, коммиты, видимость.
- Пройти проверки за 5–10 минут.

## Чеклист

### Видимость и ветка

- [ ] Репозиторий **Public**.
- [ ] Работаешь в default-ветке (обычно `main`); нужные коммиты влиты в неё.

### Локальные проверки

- [ ] `python scripts/check_links.py` — без ошибок.
- [ ] `python scripts/validate_repo.py` — без ошибок.
- [ ] `python scripts/check_markdown_hygiene.py` — без ошибок.
- [ ] `python scripts/validate_progress_ids.py` — без ошибок.
- [ ] `python scripts/validate_docs_index.py` — без ошибок.

### CI на GitHub

- [ ] Вкладка **Actions**: последний run для твоего коммита — **успешный** (зелёная галочка).
- [ ] Нет «красных» коммитов на main без исправления.

### Документация и прогресс

- [ ] README актуален (цель, навигация, как работать).
- [ ] PROGRESS.md: последние задачи с Status Done и заполненными Files changed.
- [ ] CHANGELOG при необходимости обновлён (секция Unreleased).

### Перед верификацией гильдии

- [ ] Количество коммитов на default-ветке не меньше требуемого (например 100).
- [ ] Email в коммитах совпадает с аккаунтом GitHub, который подключаешь.
- [ ] Репо не в архиве (не Archived).

## Проверка

- Все пункты отмечены — можно переходить к верификации или созданию релиза.

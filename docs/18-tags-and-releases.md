# Теги и релизы

Как создать тег, отправить его на GitHub и оформить релиз.

## Цель

- Понимать, зачем нужны теги (версии, точки для отката).
- Создать тег и связать его с Release на GitHub.

## Теги в Git

- **Тег** — именованная метка на конкретном коммите (часто для версий: v1.0.0).
- **Легковесный тег:** просто имя. **Аннотированный тег:** имя + сообщение + автор (рекомендуется для релизов).

## Создание и отправка тега

1. **Аннотированный тег на текущем коммите:**
   ```bash
   git tag -a v1.0.0 -m "Первый релиз: документация и скрипты"
   ```

2. **Тег на конкретном коммите:** `git tag -a v1.0.0 <SHA> -m "Сообщение"`.

3. **Отправить тег на GitHub:** `git push origin v1.0.0` или все теги: `git push origin --tags`.

## Релизы на GitHub

- На странице репозитория: **Releases** → **Create a new release**.
- Выбери существующий тег или создай новый при создании релиза.
- Заполни описание (что вошло в релиз), при желании приложи бинарники.
- **Publish release** — на главной репо появится бейдж последнего релиза.

## Полезные команды

- Список тегов: `git tag -l`.
- Удалить локальный тег: `git tag -d v1.0.0`.
- Удалить тег на remote: `git push origin --delete v1.0.0`.

## Проверка

- `git tag -l` — тег виден локально.
- На GitHub в разделе **Releases** отображается релиз, связанный с тегом.

## Типовые ошибки

- **Забыл запушить тег** — после `git push` без тегов релиз на GitHub не сможет указать на тег; выполни `git push origin v1.0.0`.
- **Тег на неверном коммите** — удали тег (`git tag -d`, затем `git push origin --delete`) и создай заново на нужном коммите.

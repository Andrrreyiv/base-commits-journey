# Руководство по внесению вклада

Спасибо за интерес к проекту. Кратко — как предложить изменения.

## Ветки и Pull Request

- Создавай ветку от актуального `main`: `git switch main && git pull && git switch -c feature/краткое-название`.
- Вноси изменения, коммить с понятным сообщением (см. [Стиль коммитов](docs/09-commit-messages.md)).
- Отправь ветку: `git push -u origin feature/краткое-название`.
- Открой Pull Request в репозитории на GitHub. Заполни описание и чеклист из шаблона PR.

## Стиль коммитов

- Один коммит — одна логическая правка.
- Формат: `Тип: краткое описание` (Docs / Chore / CI / Fix и т.д.).
- Подробнее: [Стиль коммитов](docs/09-commit-messages.md).

## Локальные проверки

- Перед PR запусти скрипты из `scripts/` (если есть): проверка ссылок, валидация репо, гигиена Markdown.
- Убедись, что CI на GitHub проходит (зелёный статус).

## Шаблон PR

- Опиши, что сделано и зачем.
- Укажи связанный issue (если есть): `Closes #N`.
- Отметь чеклист в шаблоне PR (что проверил).
